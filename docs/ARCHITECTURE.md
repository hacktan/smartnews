## 1. Core Philosophy

SmartNews is not a generic RSS reader. It is a **high-signal, claim-verified news discovery platform**.

The central differentiator: **cross-source claim verification**. When the same story is covered by multiple sources, SmartNews extracts specific factual claims from each article, compares them, and surfaces contradictions, agreements, and gaps. This is not a layer added on top â€” it is the foundation the product is built around.

Everything else (hype scoring, credibility ranking, entity navigation, AI briefing) supports this core mission.

## 4. Data Architecture â€” Medallion (DuckDB)

Same Bronze â†’ Silver â†’ Gold â†’ Serve pattern, now in DuckDB instead of Delta Lake.

```sql
CREATE SCHEMA bronze;   -- raw ingested data, never modified
CREATE SCHEMA silver;   -- cleaned, categorized
CREATE SCHEMA gold;     -- AI-enriched, scored, claim-extracted
CREATE SCHEMA serve;    -- UI-ready, pre-computed, full overwrite per run
```

## 5. Core Feature â€” Claim Verification

This is the primary differentiator. Every architectural decision should support this.

### How It Works

```
Story matching identifies: "3 sources covered the same story"
        â†“
04b_claim_extraction.py
  â†’ GPT reads full text of each article
  â†’ Extracts atomic factual claims:
     "Company X raised $500M" / "CEO said Y" / "Launch date is Z"
        â†“
Claim comparison across sources:
  â†’ Same claim appears in all 3? â†’ CONSISTENT
  â†’ Claim in 2 sources with different numbers? â†’ DISPUTED
  â†’ Claim only in 1 source? â†’ UNVERIFIED
        â†“
serve.claim_verdicts:
  story_id | claim_text | sources[] | verdict | confidence
```

### New API Endpoint
```

```

### New Frontend Page
